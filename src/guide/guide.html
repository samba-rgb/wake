Welcome to the Wake Guide

Your comprehensive guide to advanced Kubernetes log tailing.

Normal View of Logs

Use Wake to tail logs from Kubernetes pods in a simple, readable format. This is the default mode and provides a straightforward stream of log data.

wake -n kube-system "kube-proxy"
[2025-10-15T16:42:10Z] kube-proxy: Starting up...
[2025-10-15T16:42:12Z] kube-proxy: Syncing rules
...

TUI View of Logs

For an interactive experience, launch the Terminal UI (TUI) mode. This allows for real-time filtering, searching, and navigation within your terminal.

wake --ui

Use arrow keys to scroll, f to filter, and q to quit.

Web View of Logs

Wake can also serve a web-based UI for viewing logs, which is great for sharing and collaboration.

wake --ui --web

This will start a local web server and open the log viewer in your default browser.

Running Templates

Templates allow you to define and reuse complex log queries and output formats. This is useful for standardizing log analysis across your team.

wake --template my-template.yaml

You can specify filters, output fields, and formatting in your template file.

Running Scripts

Execute custom scripts to perform advanced log processing, such as sending alerts or integrating with other tools.

wake --script ./scripts/my_script.sh

Monitoring CPU & Memory

Wake can monitor resource usage of your pods, providing real-time insights into CPU and memory consumption.

wake --monitor

Configure monitoring options in your wake.yaml or via command-line flags.

Changing Configs

Customize Wake's behavior through a configuration file. You can specify default namespaces, filters, and output formats.

By default, Wake looks for a wake.yaml file in your current directory or home directory.
